    Same error message when trying to flash the stock firmware, so device was sort of bricked. But...

    After some investigation I found OpenOCD's scripts for the stm32f1x platform were designed to support the "official" 64k flash size only and trying to flash larger program sizes results in the above message.

    The OpenOCD documentation gives a hint how to resolve this: https://openocd.org/doc/html/Flash-Commands.html#index-stm32f1x

    This forum post (in German language) has a practical solution that worked for me: https://www.mikrocontroller.net/articles/STM32F103C8T6_STM32_Billig_Board

    What I did:

    1. in /usr/share/openocd/scripts/target/ – copy stm32f1x.cfg to stm32f103.cfg
    2. open stm32f103.cfg in an editor, search for this line:
      flash bank $_FLASHNAME stm32f1x 0x08000000 $_FLASH_SIZE 0 0 $_TARGETNAME
    3. replace with:
      flash bank $_FLASHNAME stm32f1x 0 0x20000 0 0 $_TARGETNAME
    4. in your projects openocd.cfg file, change the target to your newly saved target definition:
      source [find target/stm32f103.cfg]
    5. re-run openocd, flashing should succeed now:
